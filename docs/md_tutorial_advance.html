Hot swapping in ProjecCreationFX games

Sometimes, during a game development, it is necessary to change and test repeatedly a variable value, a character speed or an obstacle size, and the only thing you can do is close the game, open it again and check your changes. In some occasions, to test that change can take a lot of time, and it can be really annoying.

A better solution for this problem is to do changes during your game execution and see the result in real time. This can be accomplish using HotswapAgent. Its configuration is really simple: download the last compatible version, put the JAR file in a desired location, and configure the runtime environment to use it.

You just need to add the following line in VM arguments to the JVM in Run Configurations:

-XXaltjvm=dcevm -javaagent:"C:\...\hotswap-agent-X.X.X.jar"=autoHotswap=true

Remember: You have to use a JDK environment to make it work properly.
