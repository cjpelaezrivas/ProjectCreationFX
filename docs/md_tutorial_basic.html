This is an initial tutorial to learn the basic features of Project Creation FX. Following the next steps you will be able to create your first project and start using this library.

Getting started

The very first thing you have to do is download the library from the downloads page. You can choose any version, but it is recommended to get the last one. As a general rule, all tutorials and information on this page will be updated to the last version, so tutorials and wiki could not be coherent with older versions.

Once you have downloaded the library, you have to add it to your project path. It uses to be easy, in Eclipse for example, just go to Java Build Path in project properties and add the downloaded JAR in Libraries tab.

Keep in mind: Java 8 or later is needed.

Now you are ready to start coding.

Creating your first project

The best way to start is trying to create something simple. What about the simplest possible project?

Creating this project can be the best way to check that everything is working fine. This is the simplest (minimal) project you can create with Project Creation FX. From here, you only can go up .

prMinimal.java
import tk.prlazarus.projectcreationfx.engine.MainManagerAdapter;
import tk.prlazarus.projectcreationfx.engine.gamestates.GameStateManager;

public class prMinimal extends MainManagerAdapter {

    @Override
    public void statesCreation() {
        GameStateManager.Instance().addGameState("minimal", new MinimalState());
        GameStateManager.Instance().changeGameState("minimal");
    }

    public static void main(String[] args) {
        launch(args);
    }
}
MinimalState.java
import tk.prlazarus.projectcreationfx.engine.gamestates.GameState;
import tk.prlazarus.projectcreationfx.engine.graphics.GraphicsAdapter;

public class MinimalState extends GameState {

}

I know this is a black screen, but if you didn’t get any errors: it works!

OK, let’s check what have you done. In the previous example you created a main class that extends from MainManagerAdapter. This father class offers implemented all the not required methods. The only function you are forced to implement here is statesCreation(). This method is used to declare all game states the game is going to use. Also, you have to select an initial states using changeGameState() method or you will get an exception.

The state is even simpler than the main class. It only has to extend from GameState. That’s all. The GameState father class implements all the methods that are used by the engine. Naturally, you can override some methods here, but let’s leave that for later.
